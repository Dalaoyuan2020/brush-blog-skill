### ❌ 问题 1：按钮无法点击
**现象：** 博客卡片下方的按钮（👍👎📖💾🔄）点击没反应
**原因：** Telegram inline keyboard callback 未接入
**影响：** 用户无法通过按钮交互，只能用文字命令
**临时方案：** 使用 `/brush like` 等文字命令代替

### ❌ 问题 2：缺少冷启动兴趣选择
**现象：** 第一次使用直接推荐文章，没有让用户选择兴趣方向
**影响：** 推荐内容可能不匹配用户兴趣（羊爸爸反馈第一篇不感兴趣）
**建议方案：** 
- 首次使用时弹出兴趣选择界面
- 让用户从 6 个分类中选 2-3 个感兴趣的
- 根据选择初始化推荐权重

### ❌ 问题 3：首篇文章不匹配兴趣
**现象：** 第一篇推荐的是 Rust/eBPF 技术文章，用户不感兴趣
**原因：** 冷启动默认推 Hacker News 热门，没有个性化
**建议方案：** 结问题 2，通过兴趣选择解决

## 当前不足（写进 Skill 文档）
### 1. 响应速度慢
**现象：** 每次 `/brush` 需要等待数秒
**可能原因：**
- RSS 抓取网络请求超时设置过长（10 秒）
- 每次请求都刷新内容池
- 推荐算法计算开销
- Moltbot 调用 Python 脚本的进程启动开销

**临时方案：** 用户等待
**长期方案：** 
- 后台异步刷新内容池
- 缓存推荐结果
- 预加载下一篇文章

### 2. 按钮无法点击
**现象：** Telegram 按钮显示为文字，无法点击
**原因：** 技能以 CLI 方式运行，未接入 Moltbot inline keyboard 系统
**临时方案：** 使用文字命令（`/brush like` 等）
**长期方案：** 将技能集成为 Moltbot 原生技能

### 3. 深度阅读功能不完整
**现象：** `/brush read` 只显示 RSS 摘要（2-3 句话），不是完整文章
**原因：** 只读取 RSS feed 的 `<summary>` 字段，没有抓取原文链接的完整内容
**影响：** 用户体验差，"深度阅读"名不副实
**临时方案：** 用户点击原文链接手动阅读
**长期方案：** 
- 实现全文抓取（fetch + HTML parse）
- 清洗 HTML（去掉广告、导航、评论）
- 提取正文内容
- 可选：保存到本地或 Notion

### 4. AI 摘要讲解质量差
**现象：** 用户要求"用大白话讲解文章"，AI 输出的是专业术语堆砌，像"列了一些，把每句话摘了一些部分"
**原因：** 
- 只是机械地提取文章片段，没有真正理解后重新组织语言
- 缺少"翻译"能力：把专业内容转化成小白能懂的大白话
- 缺少结构化：没有用比喻、例子、场景来解释
**影响：** 用户读不懂、感觉费劲、莫名其妙
**改进方向：**
- 用比喻和类比（如"打草稿"比喻思维链）
- 用具体例子（如数学题演示）
- 结构化输出（场景→问题→解决→效果）
- 去掉专业术语，或用白话解释术语
